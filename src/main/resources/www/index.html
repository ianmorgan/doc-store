Graph Store

About

Setup DDD style aggregates from a GraphQL schema. Store aggregate state to an event-store using REST, and query using both GraphQL and traditional REST. More advanced applications can use CQRS principle to build custom data integrations and views by simply reducing the underlying event stream. Here is an article on why GraphQL is nice implementation tool for DDD. And this presentation walks through a real world project that used GraphQL as the contract between UI and backend developers. For more detail see the online docs or the Github repo.

Individual types can be accessed via a more traditional resource style API. Using this API data is returned in a simpler form, a key difference being that for any linked document just raw ID will be returned, whereas via GraphQL it will be expanded automatically into its document

Update

To update, use one of the three POST form

http://graph-store:7001/docs/Droid/2001
http://graph-store:7001/docs/Droid
http://graph-store:7001/docs

The first form in the 'correct' REST style, the other two are really for convenience. All forms accept either JSON in the body or as a FORM submission with the JSON in a field named "payload" Note that API is designed to work with document fragments, see the Docs for more detail There is a simple HTML form for basic testing.
